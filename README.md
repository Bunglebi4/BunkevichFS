# BunkevichFS

Учебная файловая система для ядра Linux 6.12, размещаемая поверх существующего
блочного устройства. Содержит две копии суперблока (с CRC32-проверкой
целостности), фиксированный набор файлов одинакового размера и набор IOCTL для
управления и диагностики.

## Состав

| Файл              | Назначение                                                |
|-------------------|-----------------------------------------------------------|
| `bnkfs.c`         | Ядерный модуль ФС (регистрация, mount, VFS, IOCTL).      |
| `bnkfs_ioctl.h`   | Общий заголовок с описанием IOCTL и структур.            |
| `user_tool.cpp`   | Userspace C++ утилита: round-trip запись/чтение + CLI IOCTL. |
| `Makefile`        | Сборка модуля и утилиты.                                  |

## Сборка

```
make
```

Получаем `bnkfs.ko` (модуль ядра) и `user_tool` (утилита).

## Параметры модуля

| Параметр           | По умолчанию | Описание                                              |
|--------------------|--------------|-------------------------------------------------------|
| `disk_name`        | `""`         | Ожидаемое короткое имя блочного устройства.           |
| `sb1_offset`       | `0`          | Сектор первой копии суперблока.                       |
| `sb2_offset`       | `16`         | Сектор второй копии суперблока.                       |
| `max_name_len`     | `32`         | Максимальная длина имени файла.                       |
| `max_file_sectors` | `4`          | Размер каждого файла в секторах (M).                  |

## Пример использования

```bash
# 1. Подготовим тестовое блочное устройство через loopback.
dd if=/dev/zero of=/tmp/bnkfs.img bs=1M count=4
sudo losetup /dev/loop0 /tmp/bnkfs.img

# 2. Загрузим модуль с нужными параметрами.
sudo insmod bnkfs.ko \
    disk_name=loop0 sb1_offset=0 sb2_offset=32 \
    max_name_len=32 max_file_sectors=8

# 3. Смонтируем ФС в /mnt. При первом mount модуль отформатирует устройство.
sudo mount -t bnkfs /dev/loop0 /mnt

# 4. Посмотрим содержимое.
ls /mnt
# file_0000  file_0001  ...

# 5. Запустим демонстрацию: запись случайного числа и чтение обратно.
sudo ./user_tool roundtrip /mnt

# 6. Получим хэши всех файлов через IOCTL.
sudo ./user_tool list /mnt

# 7. Получим маппинг секторов для конкретного файла.
sudo ./user_tool mapping /mnt/file_0003

# 8. Обнулим все файлы.
sudo ./user_tool zero /mnt

# 9. Сотрём ФС (затрёт оба суперблока).
sudo ./user_tool erase /mnt
sudo umount /mnt
# При повторном mount устройство будет переформатировано — целостность
# суперблоков нарушена, и обе копии не пройдут CRC32-проверку.
```

## Как устроена ФС на диске

```
+----------------+   sector 0:               первая копия суперблока
|   SB copy 1    |
+----------------+
|     ...        |
+----------------+   sector sb2_offset:      вторая копия суперблока
|   SB copy 2    |
+----------------+
|  file_0000     |   M секторов
+----------------+
|  file_0001     |   M секторов
+----------------+
|     ...        |
+----------------+
```

При mount модуль:
1. Читает основной суперблок, проверяет magic и CRC32.
2. При ошибке пытается прочитать копию; если она цела — восстанавливает основной.
3. Если обе копии невалидны — форматирует устройство (расставляет суперблоки и
   обнуляет данные).

## IOCTL

Все команды доступны на любом открытом файле или директории внутри ФС:

| Команда                | Действие                                                |
|------------------------|---------------------------------------------------------|
| `BNKFS_IOC_ZERO_ALL`   | Обнулить содержимое всех файлов.                        |
| `BNKFS_IOC_ERASE_FS`   | Затереть обе копии суперблока.                          |
| `BNKFS_IOC_GET_HASHES` | Вернуть имена и CRC32-хэши всех файлов.                 |
| `BNKFS_IOC_GET_MAPPING`| По имени файла вернуть стартовый сектор и их количество.|
